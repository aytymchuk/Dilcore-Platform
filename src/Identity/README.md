# Identity Domain

Handles user management at the platform level. It manages users and their relationships to tenants, including defining access levels and permissions.

## Projects

*   **Identity.Domain**: Core DDD models (Aggregates, Entities, Value Objects).
*   **Identity.Core**: Business logic and use cases.
*   **Identity.Store**: Data persistence and repositories.
*   **Identity.Infrastructure**: External service integrations.
*   **Identity.WebApi**: Minimal API endpoints.
*   **Identity.Actors.Abstractions**: Orleans actor interfaces.
*   **Identity.Actors**: Orleans actor implementations.
